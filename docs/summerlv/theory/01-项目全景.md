# 01 - OpenClaw 项目全景

## 项目定位

OpenClaw 的口号是 **"The AI that actually does things"** —— 真正能做事的 AI。

它是一个运行在你自己设备上的个人 AI 助手，通过你已有的即时通讯应用（WhatsApp、Telegram、Discord 等）与你交互，遵循你设定的规则。

> 核心差异：不是你去找 AI，而是 AI 住在了你的聊天窗口里。

## 项目演进

OpenClaw 经历了多次更名和进化：

```
Warelay → Clawdbot → Moltbot → OpenClaw
```

从个人学习 AI 的实验项目，逐步成长为开源社区驱动的 AI 助手平台。

## 核心特性

### 1. 多频道统一接入
- 支持 **21+ 即时通讯频道**：WhatsApp、Telegram、Discord、Slack、Signal、iMessage、Matrix、飞书、MS Teams、QQ Bot 等
- 一个 Gateway 进程桥接所有频道
- 你从任何频道发消息，都能得到同一个 AI 助手的响应

### 2. 多模型提供商
- 支持 **40+ AI 模型提供商**：OpenAI、Anthropic、Google、DeepSeek、Ollama（本地模型）等
- 可自由切换模型，支持故障转移
- BYO-Key（自带 API Key）模式，数据不经过第三方

### 3. 本地优先 + 隐私优先
- 运行在你自己的设备上，数据不离开你的机器
- 支持本地模型（Ollama），完全离线可用
- 安全沙箱机制，限制 AI 的文件系统和命令执行权限

### 4. 自主 Agent 能力
- 不仅仅是聊天机器人，而是能**真正执行任务**的 Agent
- 支持浏览器操控、文件编辑、代码执行、定时任务
- Heartbeat（心跳）系统：AI 可以主动思考和行动，不必等你发消息

### 5. 插件生态
- 110+ 内置扩展插件
- 完善的 Plugin SDK，第三方可开发自己的插件
- 插件涵盖：模型提供商、频道、搜索、语音、记忆、图片/视频生成等

### 6. 跨平台
- 服务端：macOS、Linux、Windows、Docker
- 客户端：macOS 桌面应用、iOS 应用、Android 应用
- 甚至可以在树莓派上运行

## 技术栈

| 层面 | 技术 |
|------|------|
| 核心语言 | TypeScript（Node.js/Bun） |
| 通信协议 | WebSocket（Gateway 线协议） |
| 模式定义 | TypeBox → JSON Schema |
| 移动端 | Swift (iOS) / Kotlin (Android) |
| 打包部署 | Docker / npm 全局安装 |
| 插件加载 | jiti（运行时 TypeScript 加载） |

> 为什么选 TypeScript？OpenClaw 本质上是一个编排系统：提示词、工具、协议、集成。TypeScript 让 OpenClaw 默认可 hack——广泛流行、快速迭代、易读易改。

## 为什么 OpenClaw 能火？

1. **自主 Agent 范式**：不是问答式聊天，而是能主动执行任务的 Agent
2. **IM-as-AI-OS**：把即时通讯变成 AI 操作系统，你不需要打开新 App
3. **本地优先 + 隐私**：数据在自己设备上，不是 SaaS
4. **开源 vs 闭源**：对比 ChatGPT/Claude 等闭源产品，完全可控
5. **Vibe Programming**：AI 辅助编程降低了贡献门槛，社区快速膨胀
