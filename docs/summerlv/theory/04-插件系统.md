# 04 - 插件系统

> 官方文档：`docs/plugins/architecture.md`（80KB）、`docs/plugins/sdk-overview.md`、`docs/plugins/building-plugins.md`

## 核心理念

**核心保持精简，可选能力以插件形式交付。**

OpenClaw 的插件系统是整个可扩展能力的基础，110+ 内置扩展全部通过这套机制加载运行。

## 插件能提供什么

| 能力类型 | 示例 |
|---------|------|
| **频道（Channel）** | Discord、Telegram、飞书、Teams |
| **模型提供商（Provider）** | OpenAI、Anthropic、DeepSeek、Ollama |
| **Web 搜索** | Brave、DuckDuckGo、Tavily |
| **语音（TTS/STT）** | ElevenLabs、Deepgram |
| **图像/视频生成** | fal.ai、ComfyUI、Runway |
| **记忆** | memory-core、memory-wiki、memory-lancedb |
| **浏览器自动化** | browser（Playwright） |
| **Agent 工具和 Hooks** | 自定义工具、事件钩子 |

## 四层加载管线

```
1. Discover（发现）
   ↓  扫描 extensions/ 目录 + 已安装的 npm 包
   ↓  读取 openclaw.plugin.json manifest

2. Validate（验证）
   ↓  拒绝不安全的 manifest
   ↓  规范化配置

3. Decide Enablement（决定启用）
   ↓  按用户配置决定是否启用
   ↓  单例能力检查（如记忆只能一个）

4. Load + Register（加载+注册）
   ↓  通过 jiti 运行时加载 TypeScript
   ↓  注册到能力注册表
   ↓  暴露给 Agent 和 Gateway
```

## Plugin Manifest（插件清单）

每个插件需要一个 `openclaw.plugin.json`：

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "description": "Adds a custom tool",
  "version": "1.0.0"
}
```

`package.json` 中声明插件入口：

```json
{
  "name": "@myorg/openclaw-my-plugin",
  "openclaw": {
    "extensions": ["./index.ts"],
    "compat": {
      "pluginApi": ">=2026.3.24-beta.2",
      "minGatewayVersion": "2026.3.24-beta.2"
    }
  }
}
```

## Plugin SDK 边界

**唯一公开跨包契约**：`openclaw/plugin-sdk/*`

**正确的导入方式：**
```typescript
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { defineChannelPluginEntry } from "openclaw/plugin-sdk/channel-core";
import { defineSingleProviderPluginEntry } from "openclaw/plugin-sdk/provider-entry";
```

**禁止**直接导入 `src/**`！

### 常用 SDK 子路径

| 子路径 | 用途 |
|--------|------|
| `plugin-sdk/plugin-entry` | 通用插件入口 `definePluginEntry` |
| `plugin-sdk/core` | 频道入口、Channel 工厂等 |
| `plugin-sdk/channel-core` | 频道插件入口 `defineChannelPluginEntry` |
| `plugin-sdk/provider-entry` | Provider 插件入口 |
| `plugin-sdk/config-schema` | 配置 Schema `OpenClawSchema` |
| `plugin-sdk/channel-setup` | 可选频道设置 Surface |
| `plugin-sdk/setup` | 设置向导助手 |

## 三种插件形态

### 1. Tool/Hook 插件（最简单）

```typescript
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";

export default definePluginEntry({
  id: "my-plugin",
  setup(api) {
    api.registerTool({
      name: "my_tool",
      description: "Does something",
      schema: { /* JSON Schema */ },
      async execute(params) {
        return { result: "done" };
      }
    });
  }
});
```

### 2. Channel 插件

```typescript
import { defineChannelPluginEntry } from "openclaw/plugin-sdk/channel-core";

export default defineChannelPluginEntry({
  id: "my-channel",
  // 处理入站消息、发送消息、连接/断开等
});
```

### 3. Provider 插件

```typescript
import { defineSingleProviderPluginEntry } from "openclaw/plugin-sdk/provider-entry";

export default defineSingleProviderPluginEntry({
  id: "my-provider",
  // catalog hook、推理 hook、streaming hook 等
});
```

## Plugin Hooks（44+ 个钩子）

Provider 插件通过 hooks 参与 Agent 生命周期：

| Hook | 时机 |
|------|------|
| `before_model_resolve` | 模型解析前，可覆盖 provider/model |
| `before_prompt_build` | Prompt 构建前，可注入上下文 |
| `before_agent_reply` | LLM 调用前，可返回合成回复 |
| `after_tool_call` | 工具执行后，可变换结果 |
| `message_received` | 入站消息 |
| `message_sending` | 出站消息前，可取消 |
| `session_start/end` | 会话边界 |
| `gateway_start/stop` | Gateway 生命周期 |

## 安装与管理

```bash
# 安装插件（优先 ClawHub，回退 npm）
openclaw plugins install @myorg/my-plugin

# 列出已安装插件
openclaw plugins list

# 查看插件详情
openclaw plugins info my-plugin

# 启用/禁用插件
openclaw plugins enable my-plugin
openclaw plugins disable my-plugin
```

## 插件分发

不需要合并到 OpenClaw 主仓库！推荐流程：

1. 发布到 [ClawHub](https://clawhub.ai)（社区首选）
2. 或发布到 npm
3. 用户通过 `openclaw plugins install <package-name>` 安装

## 关键约束

- **记忆插件**：同时只能激活一个（独占槽位）
- **版本兼容**：`compat.minGatewayVersion` 声明最低 Gateway 版本要求
- **运行时依赖**：插件依赖放在 `dependencies`，不能用 `workspace:*`
- **不要用 `workspace:*`**：npm install 时会失败
