# 05 - 频道与消息

> 官方文档：`docs/channels/index.md`、`docs/channels/channel-routing.md`、`docs/channels/groups.md`

## 频道概览

OpenClaw 支持 **25+ 消息频道**，通过 Gateway 统一接入。

### 核心内置频道（`src/channels/`）

| 频道 | 技术方案 | 备注 |
|------|---------|------|
| **WhatsApp** | Baileys | 最流行，需 QR 配对，磁盘状态多 |
| **Telegram** | grammY Bot API | **最快配置**，仅需 Bot Token，支持群组 |
| **Discord** | Discord Bot API + Gateway | 支持服务器/频道/DM |
| **Slack** | Bolt SDK | 工作区应用 |
| **Signal** | signal-cli | 隐私优先 |
| **iMessage (legacy)** | imsg CLI | 已废弃，推荐 BlueBubbles |

### 插件频道（`extensions/`）

| 频道 | 特点 |
|------|------|
| **飞书 (Feishu)** | WebSocket 连接，企业常用 |
| **MS Teams** | Bot Framework，企业级 |
| **Matrix** | 开放协议，自托管友好 |
| **Mattermost** | 开源 Slack 替代，自托管 |
| **BlueBubbles** | 推荐 iMessage 方案 |
| **QQ Bot** | QQ Bot API，私聊/群聊/富媒体 |
| **Zalo** | 越南最流行即时通讯 |
| **WeChat** | 腾讯 iLink Bot，QR 登录 |
| **IRC** | 经典 IRC 协议 |
| **LINE** | LINE Messaging API |
| **Nostr** | 去中心化 DM（NIP-04） |
| **Twitch** | Twitch 聊天（IRC） |
| **Synology Chat** | Synology NAS Webhook |
| **Nextcloud Talk** | 自托管 Nextcloud 聊天 |
| **Tlon** | Urbit 消息 |
| **Voice Call** | Plivo/Twilio 电话（需额外安装） |
| **WebChat** | Gateway 内置 Web UI |

## 频道选择建议

| 场景 | 推荐频道 |
|------|---------|
| 快速上手测试 | **Telegram**（最简单，仅需 Bot Token） |
| 个人主力使用 | WhatsApp 或 Telegram |
| 企业内部 | 飞书、MS Teams、Slack |
| 隐私优先 | Signal、Matrix |
| 开发调试 | WebChat（无需任何配置） |
| iMessage 用户 | BlueBubbles |

## 消息路由

多频道同时运行时，Gateway 按 **chat session** 路由消息：

```
入站消息
 ↓
频道插件（识别来源 chat/用户/群组）
 ↓
路由决策（按 bindings 配置）
 ↓
目标 Agent（main 或指定 agentId）
 ↓
Agent Loop 处理
 ↓
同频道回复
```

### 多 Agent 路由（bindings）

通过配置 `bindings` 把特定频道路由到特定 Agent：

```json
{
  "agents": {
    "list": [
      { "id": "work", "bindings": [{ "channel": "slack" }] },
      { "id": "personal", "bindings": [{ "channel": "telegram" }] }
    ]
  }
}
```

## DM 配对与 Allowlist

出于安全考虑，OpenClaw 不响应任意陌生人：

- **Allowlist**：仅允许特定用户/群组触发 Agent
- **DM 配对**：通过 `/pair` 命令建立信任关系
- 不在 allowlist 的消息会被静默忽略

```bash
# 查看频道状态和配对信息
openclaw channels status
openclaw channels status --probe
```

## 群组支持

不同频道的群组行为不同：

| 频道 | 群组触发方式 |
|------|------------|
| Telegram | @bot_name 或配置 trigger 词 |
| Discord | 被 @mention |
| Slack | 被 @mention 或 DM |
| WhatsApp | 被 @mention 或配置关键词 |

详见 `docs/channels/groups.md`。

## 共享 message 工具

Agent 通过统一的 `message` 工具发送消息到任何频道，插件提供特定频道的 Action：

```
Agent 调用 message 工具
 ↓
核心 message 工具（channel-agnostic）
 ↓
分发给对应频道插件的 action handler
 ↓
通过频道 SDK 发送
```

## 频道诊断与排障

```bash
# 检查所有频道状态
openclaw channels status

# 探测频道连通性
openclaw channels status --probe

# 查看频道日志
openclaw logs --channel telegram

# 频道重连
openclaw channels restart telegram
```

排障文档：`docs/channels/troubleshooting.md`
