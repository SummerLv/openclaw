# 07 - 安全与部署

> 官方文档：`docs/gateway/sandboxing.md`、`docs/gateway/secrets.md`、`docs/security/THREAT-MODEL-ATLAS.md`、`docs/install/`

## 安全设计理念

> "Security in OpenClaw is a deliberate tradeoff: strong defaults without killing capability."
> — 在强默认安全和保留能力之间做有意识的权衡

- 默认绑定 `127.0.0.1`（回环），不对外暴露
- 沙箱限制 AI 的文件系统和命令执行权限
- 密钥管理避免明文存储
- Allowlist 防止未授权访问

## 沙箱（Sandboxing）

沙箱限制 Agent（以及 Agent 调用的工具）能做什么：

### 工具策略级别

| 级别 | 说明 |
|------|------|
| **Sandbox（沙箱）** | 最严格，文件访问和命令执行受限于 workspace |
| **Tool Policy** | 中等，允许配置的工具，拒绝危险操作 |
| **Elevated（提权）** | 需要用户明确批准，执行高风险操作 |

```bash
# 查看沙箱状态
openclaw sandbox status

# 配置沙箱模式
openclaw config set agents.defaults.sandbox.enabled true
```

### Exec Approvals（命令执行审批）

Agent 执行 shell 命令时，可以要求用户批准：

```bash
# 在 CLI 中监听审批请求
openclaw approvals watch

# 审批/拒绝
openclaw approvals approve <id>
openclaw approvals reject <id>
```

## 密钥管理（Secrets）

OpenClaw 通过 **SecretRef** 系统管理敏感凭证：

```bash
# 设置密钥
openclaw secrets set OPENAI_API_KEY sk-...

# 列出密钥
openclaw secrets list

# 删除密钥
openclaw secrets delete OPENAI_API_KEY

# 应用密钥计划（批量）
openclaw secrets apply
```

**SecretRef** 在配置文件中引用密钥而非存储明文：

```json
{
  "models": {
    "providers": {
      "openai": {
        "apiKey": { "$secret": "OPENAI_API_KEY" }
      }
    }
  }
}
```

## 认证模式

### 本地访问（默认）

- 绑定 `127.0.0.1:18789`
- 共享密钥 token 认证
- 安全：只有本机进程可以连接

### 远程访问选项

| 方式 | 适用场景 |
|------|---------|
| **SSH 隧道** | 最安全，通过 SSH 端口转发 |
| **Tailscale** | 零配置 VPN，`gateway.auth.allowTailscale: true` |
| **Trusted Proxy** | 反向代理（Nginx/Caddy），`gateway.auth.mode: "trusted-proxy"` |

**警告**：不要使用 `gateway.auth.mode: "none"` 对公网暴露！

## 部署方式

### 1. 本地开发机（最常见）

```bash
# 安装
npm install -g openclaw

# 启动（开机自启）
openclaw daemon start
openclaw daemon status
```

### 2. Docker

```bash
docker run -d \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  ghcr.io/openclaw/openclaw:latest gateway
```

官方提供多个 Dockerfile：
- `Dockerfile` — 核心 Gateway
- `Dockerfile.sandbox` — 带沙箱的 Gateway
- `Dockerfile.sandbox-browser` — 带浏览器的沙箱 Gateway

### 3. VPS/云服务器

推荐搭配：
- **Hetzner**（便宜，文档详细）
- **DigitalOcean**
- **Oracle Cloud**（有永久免费层）
- **Fly.io**、**Railway**、**Render**（PaaS）

部署后通过 SSH 隧道或 Tailscale 安全访问。

### 4. 树莓派 / 家庭服务器

OpenClaw 可在树莓派 4/5 上运行，官方提供完整的 Pi 配置文档（`docs/pi.md`，21KB）。

### 5. Kubernetes

提供 Helm Chart 和 Kubernetes YAML 参考配置（`docs/install/kubernetes.md`）。

## 常用配置示例

### 最小配置（~/.openclaw/openclaw.json）

```json
{
  "models": {
    "default": "anthropic/claude-sonnet-4-5",
    "providers": {
      "anthropic": {
        "apiKey": { "$secret": "ANTHROPIC_API_KEY" }
      }
    }
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "token": { "$secret": "TELEGRAM_BOT_TOKEN" }
    }
  }
}
```

### 安全加固配置

```json
{
  "gateway": {
    "bind": "127.0.0.1",
    "auth": {
      "mode": "shared-secret"
    }
  },
  "agents": {
    "defaults": {
      "sandbox": {
        "enabled": true
      }
    }
  }
}
```

```bash
openclaw gateway status --verbose
```
