## 贡献路径

| 类型 | 流程 |
|------|------|
| **Bug 修复 / 小改动** | 直接开 PR |
| **新功能 / 架构变更** | 先在 GitHub Discussion 或 Discord 讨论 |
| **纯重构 PR** | **不接受**，除非维护者明确要求 |
| **仅修 CI 失败的 PR** | **不接受**，团队已在跟踪 |
| **插件/技能** | 发布到 ClawHub，不要加到核心 |

## PR 前必做

```bash
# 1. 本地测试
openclaw [your feature]

# 2. 构建 + 类型检查 + lint
pnpm build && pnpm check

# 3. 运行测试
pnpm test

# 4. 插件/频道改动：运行扩展测试
pnpm test:extension <extension-name>
pnpm test:contracts         # 如果改了共享 plugin/channel 面
pnpm test:contracts:channels
pnpm test:contracts:plugins
```

## PR 规则

- **最多 10 个开放 PR**（超出自动关闭）
- **每个 PR 一件事**，不混入无关改动
- **5000+ 行**的 PR 只在特殊情况下审查
- **附上截图**：UI/视觉改动必须有改前改后对比图
- 英文拼写和语法（代码、注释、文档、UI 字符串）
- **不要编辑 CODEOWNERS 覆盖的文件**，除非该 owner 明确要求

## AI 辅助 PR 欢迎！

OpenClaw 明确欢迎 AI 辅助（Codex、Claude 等）的 PR，但需要：

- [ ] PR 标题/描述标注 AI 辅助
- [ ] 说明测试程度（未测试/轻度测试/完整测试）
- [ ] 如有可能，附上 prompt 或会话日志
- [ ] 你理解代码在做什么
- [ ] 运行 `codex review --base origin/main` 并处理反馈

## 项目目录规范

### 导入边界

```
# 正确：通过 Plugin SDK
import { ... } from "openclaw/plugin-sdk/plugin-entry";

# 错误：直接导入核心
import { ... } from "../../src/agents/xxx";
```

### 检查脚本

```bash
# 检查 src/ 的扩展导入边界
node scripts/check-src-extension-import-boundary.mjs --json

# 检查 plugin-sdk 边界
node scripts/check-sdk-package-extension-import-boundary.mjs --json

# 检查测试 helper 边界
node scripts/check-test-helper-extension-import-boundary.mjs --json
```

## 开发环境搭建

```bash
# 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 安装依赖（推荐 pnpm）
pnpm install

# 开发运行（Bun）
pnpm dev

# 类型检查
pnpm tsgo

# 代码格式化和 lint
pnpm check

# 完整构建
pnpm build

# 安装 pre-commit hooks
prek install
```
