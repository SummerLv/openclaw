# 01 - 本地开发环境搭建

> 目标：从零搭建 OpenClaw 开发环境，能够本地运行、调试和测试。

## 前置条件

| 要求 | 版本 | 说明 |
|------|------|------|
| Node.js | **22+** | 必需，项目硬性要求 |
| pnpm | latest | 包管理器，项目锁定使用 |
| Git | latest | 版本控制 |
| macOS / Linux | - | 主要支持平台 |

> **提示**：推荐使用 `nvm` 或 `fnm` 管理 Node 版本。Bun 也受支持，但 pnpm 是主力工具。

## 步骤一：克隆代码

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

如果你打算贡献代码，先 Fork 再克隆你的 Fork：

```bash
git clone https://github.com/<your-username>/openclaw.git
cd openclaw
git remote add upstream https://github.com/openclaw/openclaw.git
```

## 步骤二：安装依赖

```bash
pnpm install
```

> **注意**：首次安装可能较慢，因为需要处理 workspace 依赖和 bundled plugins。如果遇到问题，可以尝试 `pnpm install --frozen-lockfile`。

也支持 Bun：
```bash
bun install
```

## 步骤三：验证构建

```bash
# TypeScript 类型检查 + 构建
pnpm build

# 代码检查 + 格式化
pnpm check
```

两个命令都通过，说明环境就绪。

## 步骤四：本地运行

```bash
# 开发模式运行 CLI
pnpm openclaw --help

# 或者用 dev 命令
pnpm dev
```

## 步骤五：运行测试

```bash
# 运行全部测试
pnpm test

# 运行单个扩展的测试
pnpm test:extension <extension-name>

# 查看可用的扩展名
pnpm test:extension --list

# 运行插件契约测试
pnpm test:contracts

# 针对性测试
pnpm test:contracts:channels
pnpm test:contracts:plugins
```

## 步骤六：IDE 配置

### VS Code（推荐）

1. 安装推荐扩展：项目根目录的 `.vscode/extensions.json` 已列出推荐扩展
2. TypeScript 版本：使用工作区版本（`node_modules/typescript`）
3. 开启 ESM 支持

### 关键配置文件

| 文件 | 作用 |
|------|------|
| `tsconfig.json` | TypeScript 配置，已设置 `experimentalDecorators: true` |
| `.vscode/settings.json` | 编辑器配置 |
| `prek` | Pre-commit hook 管理 |

## 步骤七：Pre-commit Hook

```bash
# 安装 pre-commit hook
prek install
```

Hook 会自动运行 `pnpm check`。快速提交时可跳过格式化检查：

```bash
FAST_COMMIT=1 git commit -m "fix: something"
```

> 仅在已手动运行过等效验证时使用 `FAST_COMMIT=1`。

## 常见问题

### pnpm install 失败

```bash
# 清理缓存重试
pnpm store prune
rm -rf node_modules
pnpm install
```

### Node 版本不对

```bash
# 使用 nvm 切换
nvm install 22
nvm use 22
```

### pnpm check 报错

```bash
# 自动修复格式问题
pnpm format

# 然后重新检查
pnpm check
```
