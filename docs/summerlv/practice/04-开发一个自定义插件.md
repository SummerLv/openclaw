# 04 - 开发一个自定义插件

> 目标：从零写一个 OpenClaw 插件，注册自定义 Agent 工具，并安装运行。

## 插件类型概览

| 插件类型 | 注册方法 | 适用场景 |
|---------|---------|---------|
| **Tool 插件** | `api.registerTool(...)` | 为 Agent 添加自定义工具 |
| **Channel 插件** | `api.registerChannel(...)` | 接入新的聊天平台 |
| **Provider 插件** | `api.registerProvider(...)` | 接入新的模型供应商 |
| **Hook 插件** | `api.registerHook(...)` | 监听和干预 Agent 生命周期事件 |

本次实战开发一个 **Tool 插件**，这是最简单的入门方式。

## 步骤一：创建项目结构

```bash
mkdir my-openclaw-plugin
cd my-openclaw-plugin
```

## 步骤二：创建 package.json

```json
{
  "name": "@myorg/openclaw-my-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./index.ts"],
    "compat": {
      "pluginApi": ">=2026.3.24-beta.2",
      "minGatewayVersion": "2026.3.24-beta.2"
    },
    "build": {
      "openclawVersion": "2026.3.24-beta.2",
      "pluginSdkVersion": "2026.3.24-beta.2"
    }
  }
}
```

**关键字段说明**：
- `openclaw.extensions`：声明插件入口文件
- `openclaw.compat.pluginApi`：兼容的 Plugin API 版本
- `openclaw.compat.minGatewayVersion`：最低 Gateway 版本

## 步骤三：创建 Manifest

创建 `openclaw.plugin.json`：

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "description": "Adds a custom tool to OpenClaw",
  "configSchema": {
    "type": "object",
    "additionalProperties": false
  }
}
```

**Manifest 是必需的**，即使没有配置项。OpenClaw 在加载插件代码前就会读取它。

## 步骤四：编写入口文件

创建 `index.ts`：

```typescript
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { Type } from "@sinclair/typebox";

export default definePluginEntry({
  id: "my-plugin",
  name: "My Plugin",
  description: "Adds a custom tool to OpenClaw",
  register(api) {
    // 注册一个 Agent 工具
    api.registerTool({
      name: "my_tool",
      description: "Do a thing with input",
      parameters: Type.Object({
        input: Type.String({ description: "The input to process" })
      }),
      async execute(_id, params) {
        return {
          content: [{ type: "text", text: `Processed: ${params.input}` }]
        };
      },
    });

    // 注册一个可选工具（需要用户手动启用）
    api.registerTool(
      {
        name: "my_dangerous_tool",
        description: "A tool with side effects",
        parameters: Type.Object({
          action: Type.String()
        }),
        async execute(_id, params) {
          return {
            content: [{ type: "text", text: `Executed: ${params.action}` }]
          };
        },
      },
      { optional: true }  // 用户需要在配置中手动启用
    );
  },
});
```

## 步骤五：注册事件 Hook（进阶）

```typescript
register(api) {
  // ... 工具注册 ...

  // 监听 Agent 生命周期事件
  api.registerHook(
    { before_tool_call: true },
    async (event) => {
      api.logger.info(`Tool call: ${event.toolName}`);
      
      // 阻止特定工具调用
      if (event.toolName === "shell" && event.params?.command?.includes("rm")) {
        return { block: true };
      }
      
      // 需要用户审批
      if (event.toolName === "my_dangerous_tool") {
        return { requireApproval: true };
      }
      
      return {};
    }
  );
}
```

## 步骤六：安装依赖

```bash
npm install @sinclair/typebox
```

> `openclaw/plugin-sdk/*` 在运行时由 OpenClaw 提供，不需要在 `dependencies` 中声明。可以放在 `devDependencies` 或 `peerDependencies` 中。

**重要**：避免在 `dependencies` 中使用 `workspace:*`，因为 `npm install` 会破坏它。

## 步骤七：发布与安装

### 方式一：发布到 ClawHub

```bash
# 先做干运行验证
clawhub package publish your-org/your-plugin --dry-run

# 正式发布
clawhub package publish your-org/your-plugin

# 安装
openclaw plugins install clawhub:@myorg/openclaw-my-plugin
```

### 方式二：发布到 npm

```bash
npm publish
openclaw plugins install @myorg/openclaw-my-plugin
```

### 方式三：本地路径安装

```bash
openclaw plugins install /path/to/my-openclaw-plugin
```

### 方式四：In-repo 插件（贡献者）

将插件放在 bundled workspace plugin tree（`extensions/` 目录），会自动被发现。

## 步骤八：启用插件

```bash
# 在配置中启用
openclaw config set plugins.allow '["my-plugin"]'

# 如果有可选工具
openclaw config set tools.allow '["my_dangerous_tool"]'
```

## 验证插件

```bash
# 重启 Gateway
openclaw gateway run --force

# 在对话中使用工具
# Agent 会自动识别并调用 my_tool
```

## Import 规范

```typescript
// ✅ 正确：使用聚焦的 subpath
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { createPluginRuntimeStore } from "openclaw/plugin-sdk/runtime-store";

// ❌ 错误：单体导入（已废弃，将被移除）
import { ... } from "openclaw/plugin-sdk";
```

## Pre-submission 清单

- [ ] `package.json` 包含正确的 `openclaw` 元数据
- [ ] `openclaw.plugin.json` Manifest 存在且有效
- [ ] 入口文件使用 `definePluginEntry` 或 `defineChannelPluginEntry`
- [ ] 所有 import 使用聚焦的 `plugin-sdk/<subpath>` 路径
- [ ] 内部 import 使用本地模块，不是 SDK 自引用
- [ ] 测试通过
- [ ] `pnpm check` 通过（in-repo 插件）
