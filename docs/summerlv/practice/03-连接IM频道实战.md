# 03 - 连接 IM 频道实战

> 目标：将 OpenClaw 接入你常用的聊天工具，实现跨平台 AI 对话。

## 频道全景

OpenClaw 支持 **25+** 消息频道，分两类：

| 类型 | 频道 | 特点 |
|------|------|------|
| **核心频道** | WhatsApp, Telegram, Slack, Discord, Signal, iMessage(legacy) | 内置于核心代码 |
| **插件频道** | Matrix, Feishu, MS Teams, QQ Bot, WeChat, Zalo, IRC, LINE, Mattermost, Nostr, Google Chat, Twitch, Nextcloud Talk, Synology Chat, Tlon, Voice Call, BlueBubbles | 通过 bundled plugin 提供 |

> **最快上手**：Telegram（只需 Bot Token）。**最常用**：WhatsApp（需 QR 配对）。

## 实战一：接入 Telegram

### 1. 创建 Telegram Bot

1. 在 Telegram 中找到 `@BotFather`
2. 发送 `/newbot`，按提示设置名称
3. 获得 Bot Token（格式：`123456:ABC-DEF...`）

### 2. 配置 OpenClaw

```bash
# 方式一：交互式设置
openclaw setup

# 方式二：命令行配置
openclaw config set channels.telegram.token "123456:ABC-DEF..."
```

或在配置文件中：

```json5
{
  channels: {
    telegram: {
      token: "123456:ABC-DEF..."
    }
  }
}
```

### 3. 重启 Gateway 并验证

```bash
# 重启
openclaw gateway run --force

# 检查 Telegram 连接状态
openclaw channels status --probe
```

### 4. 开始对话

在 Telegram 中找到你的 Bot，发送消息即可。首次使用需完成 DM 配对（安全机制）。

## 实战二：接入 WhatsApp

### 1. 配置

```bash
openclaw setup  # 选择 WhatsApp
```

### 2. QR 配对

1. 启动 Gateway 后，终端会显示 QR 码
2. 打开手机 WhatsApp → 设置 → 关联设备 → 扫描 QR 码
3. 配对成功后，发给自己的消息会被 OpenClaw 接收

> **注意**：WhatsApp 会话状态保存在磁盘上，重新配对较少见。

## 实战三：接入 Discord

### 1. 创建 Discord Bot

1. 访问 [Discord Developer Portal](https://discord.com/developers/applications)
2. 创建 Application → Bot
3. 开启 **Message Content Intent**（必需）
4. 获取 Bot Token
5. OAuth2 URL Generator 生成邀请链接，邀请到你的服务器

### 2. 配置

```bash
openclaw config set channels.discord.token "your-discord-bot-token"
```

### 3. Allowlist 配置

Discord 需要配置允许交互的频道和用户：

```json5
{
  channels: {
    discord: {
      token: "your-discord-bot-token",
      allowFrom: ["channel-id-1", "user-id-1"]
    }
  }
}
```

## 实战四：接入飞书（Feishu）

飞书是 bundled plugin，需要额外安装：

```bash
openclaw plugins install @openclaw/feishu
```

配置：

```json5
{
  channels: {
    feishu: {
      appId: "cli_xxx",
      appSecret: "xxx"
    }
  }
}
```

## 多频道同时运行

OpenClaw 支持同时运行多个频道：

```json5
{
  channels: {
    telegram: {
      token: "telegram-bot-token"
    },
    discord: {
      token: "discord-bot-token",
      allowFrom: ["channel-id"]
    },
    whatsapp: {}  // QR 配对
  }
}
```

消息会按来源频道路由，所有频道共享同一个 Agent 和上下文。

## 安全机制

| 机制 | 说明 |
|------|------|
| **DM 配对** | 新用户首次私聊需配对审批 |
| **Allowlist** | 限定可交互的用户/频道/群组 |
| **Group 控制** | 群组中可设置 @mention 触发策略 |
| **命令门控** | `/approve` 审批敏感操作 |

```bash
# 管理配对
openclaw channels pairing

# 审批待处理配对
openclaw channels approve <device-id>
```

## 常见问题

### Bot 无响应

```bash
# 检查频道状态
openclaw channels status --probe

# 查看 Gateway 日志
tail -f /tmp/openclaw-gateway.log
```

### Allowlist 阻止了消息

```bash
# 查看当前 allowlist
openclaw config get channels.telegram.allowFrom

# 添加允许的用户/频道
openclaw config set channels.telegram.allowFrom '["new-user-id"]'
```

### WhatsApp QR 码过期

重新启动 Gateway，会生成新的 QR 码。
