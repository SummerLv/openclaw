# 05 - 调试与问题排查

> 目标：掌握 OpenClaw 的调试方法和常见问题排查手段。

## 诊断工具

### openclaw doctor

最重要的诊断命令：

```bash
# 运行诊断检查
openclaw doctor

# 自动修复发现的问题
openclaw doctor --fix
```

Doctor 会检查：
- 配置完整性
- 插件 manifest 有效性
- API Key 配置状态
- 频道连接状态
- 依赖版本兼容性
- 遗留配置迁移

### 频道状态检查

```bash
# 检查所有频道连接状态
openclaw channels status --probe

# 查看特定频道
openclaw channels status --channel telegram
```

### 配置检查

```bash
# 查看当前运行配置
openclaw config get

# 查看特定配置项
openclaw config get models.providers
openclaw config get channels.telegram

# 验证配置（不修改）
openclaw config validate
```

## 日志分析

### Gateway 日志

```bash
# 实时查看日志
tail -f /tmp/openclaw-gateway.log

# 查看最近 120 行
tail -n 120 /tmp/openclaw-gateway.log

# 搜索错误
grep -i "error" /tmp/openclaw-gateway.log | tail -20

# 搜索特定频道日志
grep "telegram" /tmp/openclaw-gateway.log | tail -20
```

### 插件日志

插件通过 `api.logger` 输出日志，自动带上插件 ID 前缀：

```typescript
// 在插件代码中
api.logger.info("Processing request");
api.logger.warn("Something unusual");
api.logger.error("Failed to execute", error);
api.logger.debug("Debug details", { params });
```

## 常见问题排查

### 问题一：Gateway 启动失败

**症状**：`openclaw gateway run` 报错或无响应

**排查步骤**：

```bash
# 1. 检查端口占用
lsof -i :18789
# 如果被占用，杀掉旧进程
kill -9 <pid>

# 2. 检查配置
openclaw doctor

# 3. 检查 Node 版本
node --version  # 需要 22+

# 4. 尝试 verbose 模式
openclaw gateway run --verbose
```

### 问题二：频道连接断开

**症状**：Bot 不响应消息

**排查步骤**：

```bash
# 1. 检查频道状态
openclaw channels status --probe

# 2. 检查 Token 有效性
openclaw config get channels.telegram.token

# 3. 检查 allowlist
openclaw config get channels.telegram.allowFrom

# 4. 重启 Gateway
pkill -9 -f openclaw-gateway || true
nohup openclaw gateway run --bind loopback --port 18789 --force \
  > /tmp/openclaw-gateway.log 2>&1 &

# 5. 验证
openclaw channels status --probe
```

### 问题三：模型调用失败

**症状**：Agent 回复报错或无响应

**排查步骤**：

```bash
# 1. 检查 API Key
openclaw config get models.providers

# 2. 验证模型名称
openclaw config get agents.defaults.model
# 格式应为 provider/model，如 openai/gpt-4o

# 3. 检查额度/余额
# 取决于具体 Provider

# 4. 尝试切换模型
openclaw config set agents.defaults.model anthropic/claude-sonnet-4
```

### 问题四：插件加载失败

**症状**：插件不生效，Doctor 报错

**排查步骤**：

```bash
# 1. 检查插件是否安装
openclaw plugins list

# 2. 检查 manifest
cat /path/to/plugin/openclaw.plugin.json

# 3. 运行 doctor
openclaw doctor

# 4. 检查边界导入
node scripts/check-src-extension-import-boundary.mjs --json
node scripts/check-sdk-package-extension-import-boundary.mjs --json

# 5. 检查插件配置
openclaw config get plugins.entries.my-plugin
```

### 问题五：配置验证失败

**症状**：`openclaw config validate` 或 `openclaw doctor` 报配置错误

**排查步骤**：

```bash
# 1. 自动修复
openclaw doctor --fix

# 2. 检查是否有废弃配置
# Doctor 会标记 deprecated 字段

# 3. 手动检查配置文件
cat ~/.config/openclaw/openclaw.json

# 4. 检查频道配置是否与插件 manifest 匹配
# 未声明的 channels.* key 会被报错
```

## 开发调试技巧

### 本地测试扩展

```bash
# 运行单个扩展的测试
pnpm test:extension <extension-name>

# 查看可用的扩展 ID
pnpm test:extension --list

# 运行契约测试
pnpm test:contracts

# 针对性契约测试
pnpm test:contracts:channels
pnpm test:contracts:plugins
```

### 边界检查

```bash
# 检查 src/ 中的扩展导入边界
node scripts/check-src-extension-import-boundary.mjs --json

# 检查 SDK 包的扩展导入边界
node scripts/check-sdk-package-extension-import-boundary.mjs --json

# 检查测试 helper 的扩展导入边界
node scripts/check-test-helper-extension-import-boundary.mjs --json
```

### PR 提交前检查

```bash
# 完整检查
pnpm build && pnpm check && pnpm test

# 快速提交（已验证过）
scripts/committer --fast "fix: something" src/some-file.ts
```

### 使用 Codex Review

如果有 Codex 访问权限：

```bash
codex review --base origin/main
```

## 性能调试

### 内存问题

```bash
# 设置本地检查模式
OPENCLAW_LOCAL_CHECK=1 pnpm tsgo
OPENCLAW_LOCAL_CHECK_MODE=throttled pnpm lint
```

### Token 使用优化

- 检查 compaction 配置：`openclaw config get agents.defaults.compaction`
- 调整上下文窗口大小
- 使用 prompt cache 友好的模型

## 安全排查

### 密钥泄露检查

```bash
# 检查 SecretRef 配置
openclaw config get | grep -i secret

# 使用 SecretRef 而非明文
openclaw config set models.providers.openai.apiKey.secretRef "my-secret"
```

### Allowlist 审计

```bash
# 检查所有频道的 allowlist
for ch in telegram discord slack whatsapp; do
  echo "=== $ch ==="
  openclaw config get channels.$ch.allowFrom 2>/dev/null
done
```
